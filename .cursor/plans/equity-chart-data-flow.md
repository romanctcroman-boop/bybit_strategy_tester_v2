# Цепочка расчёта метрик и формирования данных для графика капитала

## 1. Движок (backend)

- **FallbackEngineV4** (`backend/backtesting/engines/fallback_engine_v4.py`):
  - На каждом баре: `equity_curve.append(equity)` — один элемент на бар.
  - Итог: массив `equity_curve` длины = число баров (например 35k за год на 15m).
- **Engine.run()** / **BacktestEngine** (`backend/backtesting/engine.py`):
  - Строит `EquityCurve(timestamps=ohlcv.index.tolist(), equity=equity_values, ...)` — **по одному значению на бар**.
  - `timestamps` = время открытия каждого бара, `equity` = капитал на конец бара.
- Метрики (MFE/MAE по сделкам) считаются в движке при закрытии сделки и попадают в `TradeRecord` (mfe, mae, mfe_pct, mae_pct). Правила расчёта единые для всех сделок.

## 2. API и формирование ответа

- **build_equity_curve_response** (`backend/api/routers/backtests.py`):
  - **Текущее поведение (исходное):** одна точка на выход из сделки (индексы по `exit_time`). Параметр `uniform_time_sampling` удалён — график возвращён в исходный вид.

## 3. Фронт (график)

- **backtest_results.js** передаёт в компонент: `timestamps`, `equity`, `trades` (с полями mfe/mae).
- **TradingViewEquityChart**:
  - Строит линию equity по парам (time, value); время в секундах (Unix).
  - Бары MFE/MAE считаются из `trades[].mfe` / `trades[].mae` в одних и тех же единицах (USD или %), нормализация через P95 и общий `maxBarHeight` — правила одни для всех баров.
  - Масштаб Y задаётся через `autoscaleInfoProvider` (equity + B&H + MFE/MAE budget).

## 4. Почему масштаб «плыл»

- Данные для графика считались **постоянно** (одни правила для всех баров и сделок).
- В API отдавалась **редкая по времени** выборка: по одной точке на каждое закрытие сделки. В начале периода могло быть много сделок (много точек подряд), в конце — мало (точки редко).
- Библиотека (LWC) подстраивала масштаб по **видимой области**. Слева на экране оказывалось много точек с большим размахом equity → крупный масштаб; справа — мало точек с меньшим размахом → мелкий масштаб. Визуально это выглядело как «разный масштаб баров».

## 5. Что сделано

- В **build_equity_curve_response** используется **одна точка на сделку** (индексы по exit_time). Равномерная выборка отключена — график в исходном виде.
