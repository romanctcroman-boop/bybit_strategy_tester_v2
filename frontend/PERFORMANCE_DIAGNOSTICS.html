🔍 Анализ причин зависания market-chart.html

📌 Диагноз: Вероятные причины зависания на 150+ секунд

  1. Memory Leak в массивах currentCandles/currentVolumes

    При долгой работе массивы могут накапливать данные, вызывая GC паузы

  2. WebSocket reconnect loop

    Если WS отключается, attemptReconnect может блокировать

  3. candleSeries.update() при большом количестве данных

    TradingView может тормозить при >2000 свечей

  4. Infinity scroll loading (loadMoreHistory)

    Может загружать огромные объёмы данных

✅ Рекомендации по исправлению

1. Добавить профилирование в handleBybitKline

function handleBybitKline(data) {
    const start = performance.now();
    
    // ... existing code ...
    
    const elapsed = performance.now() - start;
    if (elapsed > 16) {  // More than 1 frame
        console.warn('[Perf] handleBybitKline took', elapsed.toFixed(1), 'ms');
    }
}

5. Добавить мониторинг памяти

// В initStallDiagnostics добавить:
setInterval(() => {
    if (performance.memory) {
        const used = Math.round(performance.memory.usedJSHeapSize / 1024 / 1024);
        const total = Math.round(performance.memory.totalJSHeapSize / 1024 / 1024);
        console.log('[Memory]', used, 'MB /', total, 'MB');
        if (used > 500) {
            console.warn('[Memory] High memory usage!');
        }
    }
}, 10000);

6. Добавить детальный лог WebSocket

let wsMessageCount = 0;
let wsLastSecondCount = 0;

// В onmessage:
wsMessageCount++;

// Логировать каждую секунду:
setInterval(() => {
    if (wsMessageCount !== wsLastSecondCount) {
        console.log('[WS] Messages/sec:', wsMessageCount - wsLastSecondCount);
        wsLastSecondCount = wsMessageCount;
    }
}, 1000);

🛠️ Как профилировать в Chrome DevTools

  1. Открыть DevTools (F12)
  2. Вкладка Performance
  3. Нажать Record (●)
  4. Дождаться зависания
  5. Нажать Stop
  6. Искать длинные жёлтые блоки (JavaScript) или фиолетовые (Layout/Paint)

Также полезно:

  • Memory вкладка → Take heap snapshot → Искать утечки
  • Console → Фильтр по "[Perf]" или "[Diag]"
